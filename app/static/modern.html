Rate Limiting

The API includes the following rate limit headers in responses:

  • X-Rate-Limit-Limit: Maximum requests per hour
  • X-Rate-Limit-Remaining: Remaining requests in current period
  • X-Rate-Limit-Reset: Seconds until rate limit resets

For more detailed documentation, visit the OpenAPI Documentation.

API Versions

ToxidAPI supports multiple API versions for backward compatibility.

Available Versions

Current

API v2

Released: May 2025

  • Enhanced toxicity detection
  • Improved sentiment analysis
  • Redis-based rate limiting
  • API key authentication
  • Better error handling
View Documentation
Deprecated

API v1

Released: January 2025

  • Basic toxicity detection
  • Simple sentiment analysis
  • No rate limiting
  • No authentication

This version will be sunset on December 31, 2025.

Please migrate to v2 as soon as possible.

Version Migration

To use a specific API version, simply change the URL path:

POST /api/v1/analyze  # Deprecated version
POST /api/v2/analyze  # Current version

Or select the version in the dropdown when using the Analyze Text tool.

About API Keys

API keys are used to authenticate your requests to the ToxidAPI. Each API key has its own rate limit and usage tracking.

Using Your API Key

Include your API key in the X-API-Key header of your HTTP requests:

curl -X POST "https://api.toxidapi.com/api/v2/analyze"
     -H "X-API-Key: toxid_demo_key_1234"
     -H "Content-Type: application/json"
     -d '{"text": "Your text to analyze"}'

API Key Security

  • Never share your API keys in publicly accessible areas such as GitHub, client-side code, or forums.
  • Keep your API keys secure and regularly rotate them for enhanced security.
  • Generate different API keys for different environments (development, testing, production).
